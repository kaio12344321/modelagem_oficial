Diagrama de Caso de Uso:

Diagrama:

 

Esclarecimento:

·         Para a construção do meu diagrama, eu optei por destacar cada funcionalidade separadamente ao invés de incluir tudo no fluxo principal, por isso tem vários includes e extends, fiz dessa forma para não me perder no raciocínio.

·         Na maioria dos casos de uso, representei o usuário como o ator principal, porque o bibliotecário também pode realizar ações para o aluno, como locar livros.

 

Fluxos:

Identificação

UC_01

Função

Realizar Cadastro

Atores

Usuario

Fluxo Principal

Usuário clica no botão "Cadastro"

Digita as seguintes informações:

1.       Nome completo

2.       Número de identificação (Matrícula ou Id_funcionário)

3.       Email

4.       Senha

Sistema valida cadastro [FS01]

 

Identificação

[FS01]

Função

Validar Cadastro

Atores

Usuario

Fluxo Secundário

Sistema verifica se o número de identificação já foi cadastrado

Caso já tenha esse "número de identificação" registrado no sistema, então não vai ser possível realizar novamente o cadastro

Caso não tenha esse "número de identificação" registrado no sistema, então o cadastro vai ser realizado

 

Identificação

UC_02

Função

Realizar Login

Atores

Usuario

Fluxo Principal

Usuário clica no botão "Login"

Digita o "número de identificação" e a senha

Sistema verifica senha [FS02]

Caso o usuário digitou três vezes errado as informações cadastradas [FS03]

 

Identificação

[FS02]

Função

Verificar Senha

Atores

Usuario

Fluxo Secundário

Sistema compara as informações digitas com as informações cadastradas

Caso as informações forem iguais, o sistema exibe a página inicial

Caso as informações forem diferentes, o sistema exibe uma mensagem de erro e contabiliza a tentativa

 

Identificação

[FS03]

Função

Redefinir Senha

Atores

Usuario

Fluxo Secundário

Caso o sistema contabilize três vezes as informações diferentes, é exibido uma mensagem pedindo para o usuário redefinir a senha

Será enviado um link no email cadastrado, esse link fará com que ele seja redirecionado para uma nova página, para a redefinição da senha

 

Identificação

UC_03

Função

Consultar Livros Locados

Atores

Usuario

Fluxo Principal

Usuário clica no botão "Consultar Livros Locados"

Digita o "número de identificação" (Matrícula)

Sistema verifica locação [FS04]

 

Identificação

[FS04]

Função

Verificar Locacao

Atores

Usuario

Fluxo Secundário

Sistema verifica se tem algum livro locado pelo usuário

Caso tenha, isso fará com que apareça na tela os livros que o aluno locou, juntamente com as seguintes informações:

1.   Nome do livro

2.   Data da locação

3.   Data da devolução

Caso não tenha, então vai retornar uma mensagem informando que o usuário não tem nenhum livro locado

 

Identificação

UC_04

Função

Realizar Renovação

Atores

Usuario

Fluxo Principal

Usuário clica no botão "Consultar Livros Locados"

Digita o "número de identificação" (Matrícula)

Sistema verifica locação [FS04]

Selecione o livro

Clique no botão "Realizar Renovação"

Sistema verifica a disponibilidade do Livro [FS05]

Sistema realiza a renovação

Nova data de devolução registrada no sistema

 

Identificação

[FS05]

Função

Verificar Disponibilidade do Livro

Atores

Usuario

Fluxo Secundário

Sistema verifica a disponibilidade do livro

Caso o livro esteja disponível, é exibido uma mensagem na tela dizendo que o livro está disponível

Caso o livro esteja indisponível, é exibido uma mensagem na tela dizendo que o livro está indisponível

 

Identificação

UC_05

Função

Realizar Locacao

Atores

Usuario

Fluxo Principal

Usuário digita o nome do livro no campo de pesquisa

Sistema verifica existência do livro [FS06]

Usuário seleciona o livro desejado

Clica no botão "Realizar Locação"

Sistema verifica a disponibilidade do Livro [FS05]

Sistema realiza a locação

Atualiza a quantidade de livros disponíveis

Guarda a data da locação e a data de devolução

 

Identificação

[FS06]

Função

Verificar Existencia do Livro

Atores

Usuario

Fluxo Secundário

Sistema verifica se tem algum livro que esteja relacionado com a pesquisa

Caso tenha, retorna algumas informações

1. Nome do livro

2. Nome do autor

3. ISBN

4. Número de páginas

5. Data de publicação

Caso não tenha, retorna uma mensagem informando que não tem nenhum livro relacionado com a pesquisa

 

Coloquei extend no caso de uso Verificar Disponibilidade do Livro para o caso de uso Realizar Locacao, se o livro não existe, então não tem a necessidade de verificar se o livro está disponível

Identificação

FS07

Função

Verificar Pendencias

Atores

Usuario

Fluxo Secundário

Sistema verifica se o aluno tem pendência

Caso tenha, aparecerá um QR Code na tela com o valor do débito

Sistema aguarda o pagamento do cliente

Caso não tenha, retorna uma mensagem informando que o aluno não possui pendência
